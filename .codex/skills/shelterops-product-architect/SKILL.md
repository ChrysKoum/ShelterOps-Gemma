# Shelterops Product Architect

## Mission

Keep the MVP focused on one winning demo path: Private intake -> Gemma structured analysis -> internal action plan -> inventory gaps -> privacy-safe public needs board -> one-click external request.

## Rules

- Build an operations dashboard, not a chatbot.
- Prefer a small complete product over optional branches.
- Keep demo mode working even when Ollama or Supabase are unavailable.
- Treat safety, privacy, and human escalation as core product features.
- Preserve the page map from `guide.md` and route all frontend decisions through that map.

## Deliverables

Create and maintain `docs/product-map.md`, `docs/page-map.md`, `docs/demo-flow.md`, and a README MVP checklist.
